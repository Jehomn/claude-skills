# 前期资料库搭建方法论

你在执行一次系统性的前期资料库搭建。目标：将任意来源的原材料转化为结构化、可引用的知识库，供后续的 PPT/文章/报告/决策使用。

## 前置准备

### 环境确认

依赖：
- Python 3: `pip install pyyaml openai --break-system-packages`
- pdftotext（CLI）：Windows 下从 xpdfreader.com 下载，或 `choco install xpdf`

检查 `scripts/grep_synonyms.py`、`scripts/verify_gaps.py`、`scripts/lit_search.py` 可用。

### 明确研究问题

收到用户输入后确认以下信息。用户已给足够明确就不追问，直接开始：
1. 研究主题和范围
2. 输出目标（PPT / 文章 / 报告 / 知识库 / 决策）
3. 时间范围（默认不限制）
4. 已有的源材料（如果有本地的 PDF/文档）

---

## 阶段 1：定范围

明确研究边界。

| 场景 | 框架 |
|------|------|
| 临床/科学问题 | PICO：患者(P) 干预(I) 对照(C) 结局(O) |
| 技术/产品调研 | TIS：技术(T) 实现(I) 场景(S) |
| 知识体系搭建 | 领域边界 + 排除清单 + 受众/输出格式 |

**产出：** 范围声明（≤ 200 字）+ 排除清单。写到项目根目录的 `研究范围.md` 或对话中呈现让用户确认。

---

## 阶段 2：源发现

三线并行，不互相等待：

### 2.1 学术文献检索

调用 `scripts/lit_search.py`：

```bash
python scripts/lit_search.py --query "[英文检索词]" --max 30
```

或参考 `references/search-strategy.md` 手写 PubMed 检索式（需要 VPN）。

补搜指南源：WSAVA / AAHA / ACVIM / Cochrane / NICE 等（根据领域选择）。

### 2.2 本地文档枚举

```powershell
Get-ChildItem -Recurse -Include *.pdf,*.docx,*.md | Select-Object FullName
```

对 PDF 批量提取文本：

```bash
pdftotext "input.pdf" "output.txt"
```

### 2.3 网页/灰色文献

根据阶段 1 的范围声明，确定需要搜索的关键词，然后用 WebSearch 或 web-access 抓取。

**产出：** 源候选清单，含来源名称、类型（学术/书籍/网页/指南）、初步相关度判断。

---

## 阶段 3：源筛选

### 3.1 纳入/排除标准

**纳入：**
- 同行评议学术文献（RCT、观察性研究、病例系列、系统综述/荟萃分析）
- 权威机构指南（政府卫生部门、专业学会、国际组织）
- 领域标准教材/专著
- 高质量灰色文献（政府报告、行业白皮书、有署名+引用的技术文档）

**排除：**
- 纯体外实验（除非研究目的就是体外方法）
- 会议摘要（最近 6 个月 + 无全文替代的除外）
- 非同行评议的个人博客/自媒体/知乎
- 宣传材料/商业推销内容
- 无法获取全文的文献

### 3.2 Tier 分层

| 层级 | 标准 | 参与范围 |
|------|------|---------|
| Tier 1 | 核心源，与主题高度相关 | 全维度参与 |
| Tier 2 | 专科源，与部分维度相关 | 按需调用 |
| 弃用 | 经审阅后明确不相关 | 不参与 |

**产出：** 分层源列表 + 每源概览摘要。

---

## 阶段 4：全量索引

对每个 Tier 1 源 x 每个维度执行。

### 4.1 建立同义词表

在项目根创建 `synonyms.yaml`，参考 `references/synonyms-template.yaml`。每个知识点列出英文和中文变体。

### 4.2 同义词扩召回

```bash
python scripts/grep_synonyms.py synonyms.yaml "extracted_text.txt" --format markdown
```

审查匹配结果，标注每个知识点在该源中的覆盖程度和页码范围。

### 4.3 相关性评分（人工判断）

| 分数 | 标准 |
|------|------|
| 3 | 专章论述（≥ 5 页连续） |
| 2 | 部分涉及（1-4 页） |
| 1 | 间接相关（零星提及） |
| 0 | 不相关 |

评分时注意跨模块/跨维度的一致性——同样的覆盖量给同样的分。

### 4.4 反证校验

对评分 1 的源执行：

```bash
python scripts/verify_gaps.py "book_chapter.txt" --known known_topics.json
```

LLM 返回候补遗漏清单 → 人工审核 → 决定是否将候选项纳入索引。

**产出：** 结构化索引文件（Markdown 表格或 JSON），含每源 x 每维度的评分、页码范围、知识点清单。

---

## 阶段 5：精读比对

从高分源中提取原文：

```bash
pdftotext -f <start_page> -l <end_page> "book.pdf" "extract.txt"
```

### 按知识点横向对比

同一个知识点，拉出每个源的说法，做成对比表。维度：具体数据、机制解释、证据级别。

### 差异标注 `[差异]`

源之间结论冲突时：

```
[差异] 源A: 丙泊酚诱导剂量 4-6 mg/kg（依据多中心 RCT n=120）
       源B: 丙泊酚诱导剂量 6-8 mg/kg（依据小型研究 n=35）
```

**核心原则：只列证据、不做裁判。** 你不知道哪个观点在具体场景中更适用——让领域专家判断。

### 证据分级

| 级别 | 符号 | 标准 |
|------|------|------|
| 强 | ●●● | 多中心 RCT、高质量系统综述/Meta、权威指南 |
| 中 | ●●○ | 单中心 RCT、高质量观察性研究、n ≥ 30 的病例对照 |
| 弱 | ●○○ | 小型 RCT (n < 30)、病例系列、专家共识、回顾性 |

**证据强度符号必须随文说明。** 每份产出在首次出现符号处（通常为证据矩阵后的第一个区块），以引用块形式简要说明三级符号判定逻辑。该领域若无多中心 RCT，则如实注明全领域最高级别（如：兽医 CPB 领域最高为 ●●○）。

**产出：** 差异报告（含对比表 + `[差异]` 标签 + 证据分级）。

## 阶段 6：知识合成

将原材料转化为对口输出。

### 五个压缩操作

| 操作 | 做什么 | 示例 |
|------|--------|------|
| 删除 | 去除非目标范围的内容 | 犬猫限定 → 删大动物内容 |
| 合并 | 多源同一知识点 → 综合陈述 | 三本书的剂量 → 一条汇总 + 差异标注 |
| 量化 | 模糊词 → 具体数字 | "可能降低血压" → "MAP 降低 15-25%" |
| 结构化 | 列表 → 因果链 | "A, B, C" → "A → B → C" |
| 分层 | 正文（必知）+ 提示（实操）+ 差异（学术诚实） | 正文 + tip box + [差异] |

### 写作铁律

- 每条结论有出处角标，无出处不写
- 数字必须具体，不写"可能""通常""有些"
- 禁止句式："值得注意的是""大量研究表明""在当今 AI 快速发展的时代""综上所述""说白了""本质上"
- 不写"需要更多研究"，写"目前最缺 X 类型的证据"
- 搜不到的信息写"该信息暂缺"，不编造
- 中文引号「」，不用 ASCII "
- **缩写规范**：医学中文写作中，任何缩写必须在全文首次出现时写"全称（English Full Name, 缩写）"，之后方可单独使用缩写。格式：「体外循环（cardiopulmonary bypass, CPB）」→ 此后用 CPB

### 文献全量保留

- 检索命中的所有 Tier 1/2 源必须全量保留在最终报告中
- 产出前逐条比对检索日志命中列表与来源表，确保无遗漏。遗漏一条即为未完成

### 输出格式

内容先写成 **Markdown**（可审阅、可搜索、可追溯），再根据目标转化为最终格式。

**检索过程记录不进入最终输出。** 检索日志和脚本问题记录是内部过程产物，仅供调试和方法论改进，不写入面向读者的最终文档（Markdown/PDF/PPTX）。

| 目标输出 | 后续 skill |
|----------|-----------|
| PPT 课件 | **ppt-builder**（父技能，本技能为其子技能） |
| PDF 报告 | vet-lit-review / bb-analysis / pdf |
| 网页/文章 | frontend-design / Typora 编辑 |

**产出：** 内容稿（Markdown）→ 最终输出。

## 验证链

```
原始材料 → 提取/检索 → AI精读 → AI写作 → 专家审阅 → 定稿
  ↑                                                      ↑
  可随时回溯验证                                        最终仲裁者
```

三个前提：源可访问、提取可复现、最终仲裁者是领域专家（不是你，不是 AI——是用户）。

---

## 并行研究条件

多个维度/模块可以并行执行阶段 2-5，当且仅当：
- 共享同一套源材料
- 输出格式相同
- 任务间无顺序依赖

有顺序依赖时必须串行（如先做"病因分析"才能做"治疗方案比对"）。

## 陷阱速查

| 陷阱 | 原因 | 解决方案 |
|------|------|---------|
| 同义词漏判 | 关键词匹配只搜到一个变体 | 建 synonyms.yaml + 用 grep_synonyms.py |
| 低分源遗漏 | 评分 1 的书被放弃得太早 | 反证校验 verify_gaps.py |
| 标准漂移 | 不同模块评分尺度不一致 | 阶段 4 开始前先定评分锚点（3 分长什么样、2 分长什么样） |
| AI 偷懒 | 大文本跳读漏关键信息 | 强制 pdftotext 分段提取 + 同义词全量 grep，不依赖 AI 逐页读 |
| 证据不可追溯 | 写了结论找不到出处 | 每条结论角标 + 阶段 4 索引记录页码范围 |
| 编造信息 | AI 凭记忆补全不会的数据 | "该信息暂缺"是合法输出 |
| 文献选择性遗漏 | 命中 20 篇但只写了感觉"最重要"的 8 篇 | 检索日志逐条比对来源表，Tier 1/2 全量保留 |
| 缩写无定义 | 首次出现时直接用了缩写（CPB、PEEP 等） | 全文扫描每个缩写的首次出现，确保全称+缩写已给出 |

## 质检清单

交付前逐条自检：

- [ ] 范围声明 + 排除清单是否明确？
- [ ] 源发现是否三线并行（学术/本地/网页）？
- [ ] 纳入/排除标准是否写下来并执行？
- [ ] Tier 分层是否合理？
- [ ] synonyms.yaml 是否覆盖了所有核心知识点？
- [ ] grep_synonyms.py 是否对每个 Tier 1 源执行？
- [ ] 评分标准是否在开始前定了锚点？
- [ ] verify_gaps.py 是否对评分 1 的源执行？
- [ ] 差异标注是否只列证据不裁判？
- [ ] 每条结论是否有出处？
- [ ] 写作是否触犯了禁止句式？
- [ ] 缩写是否全部在首次出现时定义了全称？
- [ ] 来源表是否包含所有检索命中的 Tier 1/2 文献（逐条比对检索日志）？
- [ ] 最终输出是否符合阶段 1 定义的目标格式？
